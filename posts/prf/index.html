Psuedo Random Functions

A simple explanation on what Psuedo Random numbers are, and how we can securely send information using PRF.

Introduction

In a lot of computational and mathematical programs, random numbers are important. Might be just to take probability, to shuffle a deck of cards or to simulate a game. The only way to get a number which is truly random is to depend on certain physical phenomenon which is unpredictable, like radioactive decay of isotopes or airwave static. The issue with this is, if we need large number of truly random numbers, or if we want to reproduce a result (and thus reproduce the random numbers occuring in a particular order), it is cumbersome to depend on physical variables. Hence, the concept of psuedo-random numbers arise.

Pseudo Randomness

Psuedo Random Numbers are software generate random numbers, and are close to indistinguishable from truly random numbers as possible (ofcourse there is a better mathematical expression for this :p). Before we go to the mathematical version of all these, just how do psuedo random number solve the problems we have with truly random numbers? Since they are generated using a polynomial time algorithm, it is easy to generate numerous random numbers in seconds. Also, since they are deterministic in nature, depending on a seed we can reproduce any results that we want. Ofcourse, they are not as safe as truly random numbers, but there are viable ways to make it more secure!

Let us define a function $f$, which takes in input a number $x$ and outputs $1$ if the number belongs to a uniform distribution (truly random sampling) and $0$ if otherwise. Then,

$$ Pr[(f(x^{PR}) = 1) - (f(x^{U}) = 1)] < negl(n) $$

That is, the probability of $f$ giving correct output given truly randomly sampled number from a uniform distribution $U$, is not better than probability of $f$ giving out wrong output for psuedo random number by more than $negl(n)$.
